## 💡 建议借鉴的功能

### 1️⃣ 系统状态监控（优先级：高）

**旧前端特色**：
- MongoDB 连接状态
- 飞书 Webhook 状态
- 监控股票数量
- 今日采集数量
- 爬虫最后活跃时间

**建议**：在新前端仪表盘添加系统状态卡片，让用户了解系统健康状况。

```html
<!-- 建议添加到仪表盘 -->
<div class="widget status-card">
    <h3>⚙️ 系统状态</h3>
    <div>MongoDB: ✅ 已连接</div>
    <div>今日采集: 1,234 条</div>
    <div>最后更新: 10分钟前</div>
</div>
```

---

### 2️⃣ 数据可视化图表（优先级：高）

**旧前端特色**：
- 今日预警分布饼图
- 近7日情绪走势折线图

**建议**：在新前端仪表盘添加可视化图表，提升数据展示效果。

---

### 3️⃣ 推送统计（优先级：中）

**旧前端特色**：
- 红色预警数量
- 绿色利好数量
- 蓝色动向数量
- 静默处理数量

**建议**：在仪表盘添加今日预警统计卡片。

---

### 4️⃣ 飞书测试（优先级：中）

**旧前端特色**：
- 一键发送测试消息
- 验证 Webhook 配置

**建议**：在设置页面添加"消息推送测试"功能。

---

### 5️⃣ 调度配置（优先级：中）

**旧前端特色**：
- 设置研报自动生成时间
- 仅工作日执行开关

**建议**：在设置页面添加调度任务配置。

---

### 6️⃣ 数据导出（优先级：中）

**旧前端特色**：
- 一键导出 JSON
- 一键导出 CSV

**建议**：在资讯中心/研究报告页面添加导出按钮。

---

### 7️⃣ 账号保险箱（优先级：低）

**旧前端特色**：
- 托管第三方平台账号
- AES-256 加密存储
- 支持雪球、今日头条、东方财富等

**建议**：此功能较敏感，可考虑放在设置页面或独立管理后台。

---

### 8️⃣ 视觉采集控制（优先级：低）

**旧前端特色**：
- 视觉采集状态监控
- 手动触发采集
- 成功/失败统计

**建议**：此为运维功能，可放在独立的"管理后台"中。

---

### 9️⃣ 白名单关键词管理（优先级：低）

**旧前端特色**：
- 管理新闻入库关键词
- 添加/删除关键词

**建议**：可在设置页面添加关键词管理功能。

## 🏗️ 建议的改进方案

### 方案一：增强设置页面

将运维功能整合到设置页面：

```
设置页面
├── 📊 系统状态
├── 📨 推送测试（飞书）
├── ⏰ 调度配置
├── 🔑 关键词管理
└── 📡 API 文档入口 ✅ 已添加
```

---

### 方案二：添加管理后台入口

新建一个"管理后台"页面，集中运维功能：

```
管理后台 (/admin.html)
├── 📊 系统状态
├── 🔍 搜索引擎控制
├── 🔐 账号保险箱
├── 👁️ 视觉采集控制
├── 📨 推送测试
├── ⏰ 调度配置
└── 🔑 关键词管理
```

---

### 方案三：仪表盘增强（推荐）

在现有仪表盘增加关键监控信息：

```diff
  仪表盘
  ├── 市场概览（现有）
  ├── 我的订阅（现有）
  ├── 最新资讯（现有）
+ ├── 📊 系统状态卡片
+ ├── 📈 预警统计图表
+ └── 📉 情绪走势图表
```

## ✅ 总结

| 优先级 | 建议 | 工作量 |
|--------|------|--------|
| **高** | 仪表盘添加系统状态卡片 | 1小时 |
| **高** | 仪表盘添加可视化图表 | 2小时 |
| **中** | 设置页面添加调度配置 | 1小时 |
| **中** | 设置页面添加推送测试 | 1小时 |
| **中** | 资讯页面添加数据导出 | 0.5小时 |
| **低** | 新建管理后台页面 | 4小时 |

---

## 🎯 核心建议

1. **仪表盘是重点**：借鉴旧前端的系统状态和可视化图表，让用户一目了然
2. **设置页面要充实**：将调度、推送、关键词等配置功能集中在设置页
3. **保持专业定位**：新前端是"投研终端"，运维功能可适当精简或隐藏
4. **数据导出很实用**：JSON/CSV 导出功能对用户价值高，建议添加
