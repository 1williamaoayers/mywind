# 反省书：不检查现有依赖就盲目安装

**日期**: 2026-01-01 15:29
**问题**: 没有先检查package.json就执行npm install
**浪费时间**: 约20分钟

---

## 我的错误

在实施"表格PDF提取"功能时：

1. **没有检查现有依赖**：直接执行 `npm install pdf-table-extractor`
2. **等了20分钟**：npm下载一直卡住
3. **用户提醒后才发现**：`pdf-table-extractor@1.0.3` 早就在 package.json 里了

---

## 错误分析

| 应该做的 | 我做的 |
|---------|-------|
| 先 `cat package.json` 检查依赖 | 直接 `npm install` |
| 确认包不存在再安装 | 假设需要安装 |
| 30秒完成检查 | 浪费20分钟等待 |

---

## 根本原因

1. **急于执行**：用户说"执行"后，我立刻开始安装，没有做基本检查
2. **没有看现有代码**：如果仔细看package.json，一眼就能看到依赖已存在
3. **违反KISS原则**：最简单的方法是先检查，而不是先安装

---

## 改进承诺

1. **安装任何npm包之前，必须先检查package.json**
2. **执行任何命令之前，先问自己：有没有更简单的方法确认？**
3. **不要假设，要验证**
